# 📷 Prompt: قراءة الباركود بالكاميرا (هاتف + حاسوب)

برومت جاهز للنسخ لإضافة ميزة قراءة الباركود بالكاميرا في أي تطبيق آخر — يعمل على الهاتف
والحاسوب، مع كل الدروس المهمة (خصوصاً تجنّب تجمّد التطبيق).

---

## 0) الهدف
أضِف مكوّن **ماسح باركود بالكاميرا** قابلاً لإعادة الاستخدام، يعمل على:
- **الهاتف (أندرويد عبر Capacitor)** → كاميرا أصلية عبر **ML Kit** (سريعة ودقيقة).
- **الحاسوب/المتصفح** → كاميرا الويب عبر **ZXing** (قراءة من عنصر `<video>`).
- مع **إدخال يدوي** احتياطي في كل الحالات.

واجهة الاستخدام بسيطة:
```tsx
<BarcodeScanner onScan={(code) => {/* استعمل الكود */}} onClose={() => {/* أغلق */}} />
```

## 1) الحزم المطلوبة
- أندرويد: `@capacitor/core`, `@capacitor-mlkit/barcode-scanning`
- الويب/سطح المكتب: `@zxing/browser`, `@zxing/library`
- (اختياري) `barcode-detector` كـ polyfill، و`jsqr` لرموز QR.

## 2) اكتشاف المنصة
استخدم `Capacitor.isNativePlatform()`:
- **true** (أندرويد) → مسار ML Kit الأصلي.
- **false** (متصفح/Electron) → مسار ZXing عبر `getUserMedia` + `<video>`.

## 3) السلوك على الأندرويد (ML Kit)
نمطان:
- **مسح مستمر (inline):** `BarcodeScanner.addListener("barcodesScanned", ...)` ثم
  `BarcodeScanner.startScan({ formats })`. الكاميرا الأصلية تُعرض **خلف** الـWebView،
  لذا تجعل واجهة الويب **شفافة** بإضافة صنف إلى `body`.
- **حوار كامل الشاشة (أسرع):** `BarcodeScanner.scan({ formats })` — يفتح ماسح Google
  جاهزاً ويعيد النتيجة (لا يحتاج جعل الواجهة شفافة).
- اطلب إذن الكاميرا: `checkPermissions()` ثم `requestPermissions()`.
- ثبّت وحدة Google عند الحاجة: `isGoogleBarcodeScannerModuleAvailable()` /
  `installGoogleBarcodeScannerModule()`.

**جعل الواجهة شفافة للكاميرا (للمسح المستمر):** عبر CSS:
```css
body.barcode-scanner-active { visibility: hidden; }
body.barcode-scanner-active .barcode-scanner-modal { visibility: visible; }
```
أضِف `barcode-scanner-active` على `body` عند بدء المسح، وأعطِ نافذة الماسح صنف
`barcode-scanner-modal` لتبقى ظاهرة فوق الكاميرا.

## 4) السلوك على الحاسوب/المتصفح (ZXing)
- اطلب الكاميرا عبر `navigator.mediaDevices.getUserMedia({ video: { facingMode: "environment" } })`.
- اعرض البثّ في `<video autoPlay playsInline muted>`، وشغّل قارئ ZXing على العنصر
  لاكتشاف الباركود من الإطارات.
- أوقف المسارات (`stream.getTracks().forEach(t => t.stop())`) عند الإغلاق.

## 5) واجهة المكوّن (Modal)
- نافذة بتبويبين: **"كاميرا"** و **"إدخال يدوي"**.
- تبويب الكاميرا: مستطيل معاينة بإطار/خط مسح متحرك، حالات: `loading / ready / error`،
  وزر "إعادة المحاولة".
- تبويب الإدخال اليدوي: حقل أرقام يُرسِل **تلقائياً** عند اكتمال طول صحيح (8/12/13 رقماً)
  أو عند Enter.
- زر إغلاق (X) والنقر على الخلفية يُغلق.

## 6) منطق المسح
- **حارس ضد التكرار:** `scannedRef` يمنع معالجة أكثر من قراءة (أوقف الجلسة فور أول نتيجة
  ثم نادِ `onScan` و`onClose`).
- **تطبيع الأرقام:** حوّل الأرقام العربية-الهندية (٠١٢٣) إلى لاتينية (0123) قبل الإرجاع.
- **تحقّق من الصيغة:** اقبل صيغ البقالة (EAN-13, EAN-8, UPC-A/E, Code-128, Code-39, QR)
  وتحقّق من الطول/الصحة.

## 7) ⚠️ نقاط حرجة لتجنّب تجمّد التطبيق (دروس مهمة جداً)
1. **صنف `barcode-scanner-active` يخفي التطبيق بالكامل** على الأندرويد — إن بقي عالقاً
   يبدو التطبيق متجمّداً ولا يمكن النقر/الكتابة. أزِله في **كل** المسارات:
   - عند نجاح القراءة، عند الإغلاق، **وفي catch إن فشل بدء المسح**.
   - في **تنظيف unmount** للمكوّن (ضمان نهائي مباشر:
     `document.body.classList.remove("barcode-scanner-active")`).
   - **شبكة أمان عامة** في التخطيط: أزِل الصنف عند كل تنقّل وعند عودة التطبيق للواجهة
     (`visibilitychange` → visible) — لكن لا تُزِله أثناء مسح نشط فعلاً.
2. **غلّف كل خطوات الإيقاف بـ try/catch مستقلة** حتى لا يمنع فشل `stop()` إغلاق النافذة
   أو إزالة صنف الإخفاء.
3. أوقف مستمعات ML Kit (`removeAllListeners` + `stopScan`) ومسارات الكاميرا في الويب
   عند الإغلاق (منع تسرّب الكاميرا).
4. عالج الأخطاء بوضوح: إذن مرفوض، كاميرا غير متاحة، ML Kit غير مدعوم → اعرض رسالة
   + اقترح الإدخال اليدوي.

## 8) Props والعقد
- `onScan(code: string)` — يُستدعى مرة واحدة بكود مُطبَّع وصحيح.
- `onClose()` — لإغلاق المكوّن.
- يستدعي المكوّن `onScan` ثم `onClose` تلقائياً بعد أول قراءة ناجحة.

## 9) معايير القبول
- يقرأ الباركود بالكاميرا على **الهاتف** (ML Kit) و**الحاسوب** (ZXing)، مع **إدخال يدوي**
  يعمل في الحالتين.
- يُعيد دائماً أرقاماً لاتينية صحيحة.
- **لا يتجمّد التطبيق أبداً** بعد فتح/إغلاق الكاميرا أو عند فشل المسح أو رفض الإذن
  (الصنف المخفي يُزال في كل الحالات).
- يطلب إذن الكاميرا بشكل صحيح، ويوقف الكاميرا تماماً عند الإغلاق.
